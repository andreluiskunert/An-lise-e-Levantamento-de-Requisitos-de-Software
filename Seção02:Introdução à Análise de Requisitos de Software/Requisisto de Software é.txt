Primeiro de tudo, os requisitos funcionais.

Eles descrevem as funcionalidades e tarefas específicas que o software deve executar.

Como assim? Ou um sistema de gerenciamento de

pedidos pode ter o requisito funcional de

permitir que os usuários cadastrem novos pedidos.

Então, assim, de uma forma simplificada, o que é o requisito funcional? O

requisito funcional é aquilo que vai, por obrigatoriedade ter que existir para

que o sistema ou que a ferramenta ou que o programa exista.

Ou seja, a função principal que ele precisa ter para ele poder existir.

Então, o requisito funcional é um requisito que é obrigatório.

Então, novamente dando um exemplo aqui, se um sistema faz

gerenciamento de pedidos, o que precisa haver dentro desse

sistema para que haja gerenciamento de pedidos? Premissa Cadastro

de novos pedidos? Então, isso aqui é uma regra.

Não tem como eu criar um sistema de gerenciamento de pedidos se eu não conseguir fazer

o cadastro dos meus pedidos para após eu fazer o gerenciamento desses cadastros que foram realizados.

Então isso é um requisito funcional.

Beleza é um requisito que é obrigatório existir para que

o meu sistema, minha ferramenta, o meu serviço, ele aconteça.

Já o meu requisito não funcional.
